Callbacks

DuxTree supports some optional callbacks at the tree level and the node level.

DuxTree Callbacks

These callbacks are at the tree level so they are passed as properties to the DuxTree component.

onExpand

This is called when a folder is expanded by the user. The ID of the node is passed to the callback. NOTE: if an onExpand callback is specified for the node, the tree-level callback is not called.

onCollapse

This is called when a folder is collapsed by the user. The ID of the code is passed to the callback. NOTE: if an onCollapse callback is specified for the node, the tree-level callback is not called.

onSelectionChanged

This is called when the user checks or un-checks a node or a folder of nodes. An array of the selected IDs is passed in.

Node Callbacks

These callbacks are at the node level. They are passed as properties of the node objects in the array passed to the data property of the DuxTree component.

onExpand

This is called when a folder is expanded by the user. The ID of the node is passed to the callback.

onCollapse

This is called when a folder is collapsed by the user. The ID of the code is passed to the callback.

onLoadChildren

This is called when a folder is expanded by the user. The ID of the node is passed to the callback. See the topic on Loading on Demand for more information.
