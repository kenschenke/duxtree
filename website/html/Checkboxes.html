Checkboxes

Nodes can be rendered with checkboxes if the checkable property is true. The default is false and can be specified per node in the JavaScript array or for the entire tree for the DuxTree component. The defaultChecked property on a node specifies if the node should be checked when that node is initialized.

For the purpose of this discussion a node is defined as an item with no children. A folder is defined as an item with children consisting of nodes and/or additional folders. Nodes with checkboxes are bi-state, either checked or unchecked. Folders are tri-state: checked, unchecked, or indeterminate. If a folder is checked, that means all descendant nodes are also checked. If a folder is unchecked, all descendant nodes are also unchecked. Indeterminate indicates some descendant nodes are checked, but not all. If the user interacts with the folder's checkbox, that changes the check state of all descendants. If the folder is unchecked or indeterminate, all descendant nodes are checked. If the folder is checked, all descendant nodes are unchecked.

Checkbox Rendering

By default, DuxTree renders checkboxes using HTML <input> tags. Of course, this affects appearance in different browsers. If desired, you can specify a JSX component to use instead. This is done with the checkboxChecked, checkboxUnchecked, and checkboxIndeterminate properties to DuxTree. Here is an example using Google's Material Icons: 

    <DuxTree
        data={treeData}
        checkable={true}
        checkboxUnchecked={<i className="material-icons">check_box_outline_blank</i>}
        checkboxChecked={<i className="material-icons-outlined">check_box</i>}
        checkboxIndeterminate={<i className="material-icons-outlined">indeterminate_check_box</i>}
     />
