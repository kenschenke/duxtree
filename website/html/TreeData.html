Populating a Tree

Trees are populated by a JavaScript array supplied in the data property of the DuxTree component. The array consists of objects representing each node in the tree. Because the tree is specified via a property of DuxTree, modifying the array will modify the tree. Here is a simple two-node tree. 

    const treeData = [
       {
          id: 'node1',
          label: 'Title For Node 1'
       },
       {
          id: 'node2',
          label: 'Title for Node 2'
       }
    ];

The id and label properties are required for every node. The id must be unique and can be a string or number. The label can be a string, JSX tag, or a function. The section on customizing appearance goes into more detail on labels.

Node Children

Nodes can have a children property that contains an array of child nodes. Children can be lazy-loaded via a callback. See the section on loading on demand for details. Here is an example of a node with children. The defaultExpanded property controls whether or not a node's children are shown when the tree is rendered. The default is false. 

    const treeData = [
       {
          id: 'folder1',
          label: 'Node With Children',
          defaultExpanded: true,
          children: [
             {
                id: 'child1',
                label: 'Child 1 Title'
             },
             {
                id: 'child2',
                label: 'Child 2 Title'
             }
          ]
       }
    ];
